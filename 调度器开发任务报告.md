# 调度器开发任务报告

## 结果

已完成本轮 404 修复、验证与复审。当前运行态中，`/api/channel_scheduler/*` 和兼容别名 `/api/channel-scheduler/*` 均会进入鉴权链路，未登录时返回 401，不再返回 404。

## 本轮问题

截图里的调度器入口与调度日志页仍弹出 `Request failed with status code 404`。当前新前端代码请求的是标准路径 `/api/channel_scheduler/*`，运行态验证该路径已不是 404；继续扫描发现 hyphen 旧命名 `/api/channel-scheduler/*` 仍返回 404。由此判断，浏览器缓存、旧前端包或某个旧调用分支可能仍在请求 hyphen 路径。

## 改动

- `router/api-router.go`：调度器 API 同时注册标准 `/api/channel_scheduler` 与兼容 `/api/channel-scheduler` 两个前缀，共用同一套 controller、Admin/Root 鉴权与行为逻辑。
- `router/api_router_test.go`：新增路由表回归测试，覆盖日志、统计、临时禁用列表、全局配置、渠道配置、恢复、日志删除等调度器路由，确保 hyphen 兼容路径不会再退回 Gin 404。
- `调度器开发TODO.md`：补充 T16 与本轮进度日志，记录根因假设、修复点和验证结果。

## 验证
