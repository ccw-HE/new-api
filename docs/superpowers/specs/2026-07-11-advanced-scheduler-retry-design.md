# 高级调度器重试与故障转移设计

## 目标

让高级调度器使用系统设置中的“自动重试状态码”决定是否继续尝试当前渠道；对于未配置为重试、但可归因于上游瞬时故障的响应，立即临时禁用当前渠道并切换其他渠道，而不是直接向用户返回错误。

同时完善高级调度器弹窗：统一名称为“高级调度器”，修复输入框焦点边框被裁切和横向滚动问题，并保证桌面端与移动端完整显示。

## 范围

- 仅调整高级调度器接管请求时的失败决策和调度行为。
- 旧调度器继续保持原有重试语义和 `504/524` 安全排除规则。
- 复用现有 `AutomaticRetryStatusCodes`、渠道级失败阈值、临时禁用秒数和自动恢复能力。
- 不新增第二份调度器状态码配置。
- 不提交 Git。

## 后端设计

### 四态失败决策

高级调度器不再只使用 `retryable bool`，而是使用明确的失败处置类型：

1. `stop`：停止调度并返回当前错误。
2. `retry_current`：继续按现有失败阈值尝试当前渠道；达到阈值后临时禁用并切换渠道。
3. `failover_without_disable`：立即排除当前渠道并切换其他渠道，但不修改渠道持久化状态。
4. `failover_now`：不再尝试当前渠道，立即临时禁用并切换其他渠道。

处置结果需要写入调度日志元数据，便于区分“停止”“阈值重试”和“立即故障转移”。

### 决策顺序

按以下顺序判断，前面的终止条件优先：

1. 请求上下文已经取消或超时：`stop`。
2. 命中渠道亲和规则且设置失败后不重试：`stop`。
3. 错误带有显式 `skipRetry`：`stop`。
4. 已达到单请求最大尝试次数：`stop`。
5. 请求指定了固定渠道：`stop`。
6. HTTP `2xx`：`stop`。
7. `ErrorCodeDoRequestFailed`：`failover_without_disable`。
8. 渠道内部错误、空响应等现有可重试特殊错误：`retry_current`。
9. 有效 HTTP 状态码位于 `AutomaticRetryStatusCodes`：`retry_current`。
10. 状态码不在配置中，但属于可归因于瞬时上游故障的 `408`、`429` 或 `5xx`：`failover_now`。
11. 其他错误：`stop`。

高级调度器读取配置范围时不应用旧调度器对 `504/524` 的硬编码排除。因此：

- 用户将 `504/524` 加入自动重试状态码时，高级调度器按失败阈值重试当前渠道。
- 用户未配置 `504/524` 时，高级调度器立即临时禁用当前渠道并切换其他渠道。

### 立即故障转移

`failover_now` 的处理顺序：

1. 记录本次失败和总尝试次数。
2. 立即将当前渠道从本次会话候选中排除。
3. 使用渠道级 `scheduler_auto_disable_seconds`；未配置时使用全局 `auto_disable_seconds`。
4. 调用现有原子临时禁用逻辑，更新渠道状态、到期时间、状态原因、能力缓存和通知。
5. 继续选择同优先级其他渠道；同级耗尽后按配置降级。

立即禁用继续遵守现有控制项：

- `scheduler_enabled=false`：仅从当前会话排除，不修改数据库状态。
- `respect_auto_ban=true` 且渠道 `auto_ban=false`：仅从当前会话排除，并记录观察日志。
- 渠道已被并发请求或管理员禁用：不覆盖已有状态。

### 网络连接失败

`ErrorCodeDoRequestFailed` 使用 `failover_without_disable`，保持现有行为：当前请求内排除该渠道并切换其他渠道，但不立刻写入临时禁用状态。这样可以避免代理节点自身网络抖动或 DNS 问题误伤渠道。

## 前端设计

### 名称

- 高级调度器弹窗标题改为“高级调度器”。
- 渠道页入口的辅助文本、ARIA 标签和提示同步使用“高级调度器”。
- 新增 `Advanced Scheduler` i18n 键并补齐所有支持语言。

### 响应式布局与滚动

- 全局配置滚动容器增加少量左右内边距，确保输入框焦点环和边框不会被裁切。
- 滚动容器强制 `overflow-x-hidden`，只允许纵向滚动。
- 配置表单所有输入控件保持 `min-w-0` 和 `w-full`，不使用会撑开容器的固定宽度。
- 抖动输入在窄屏下换行或使用稳定网格，不产生横向溢出。
- 桌面端保持当前弹窗宽度和信息密度；移动端使用可用视口宽度，标题、标签、说明和按钮均可正常换行。
- 底部保存按钮始终位于滚动区域之外，不随表单横向移动。

## 测试设计

### 后端

- `504` 未配置：返回 `failover_now`。
- `504` 已配置：返回 `retry_current`。
- `524` 未配置：立即临时禁用，使用渠道级禁用秒数，并切换其他渠道。
- `400` 未配置：返回 `stop`，不禁用渠道。
- `408/429/5xx` 未配置：返回 `failover_now`。
- 显式 `skipRetry`、请求取消、固定渠道和渠道亲和禁重试：返回 `stop`。
- `DoRequestFailed`：排除并切换，但不临时禁用。
- `scheduler_enabled`、`respect_auto_ban`、并发禁用保护继续生效。

### 前端

- TypeScript 类型检查通过。
- 前端生产构建通过。
- 桌面和移动视口截图检查：无横向滚动条，输入框和焦点边框完整，保存按钮可见且不重叠。

## 验收标准

- 高级调度器对状态码的当前渠道重试行为由 `AutomaticRetryStatusCodes` 决定。
- 未配置重试的 `504/524` 会立即临时禁用当前渠道，并继续尝试其他渠道。
- 请求方错误不会错误禁用渠道。
- 临时禁用时长、自动恢复、渠道级覆盖和既有安全控制保持有效。
- 弹窗标题显示“高级调度器”。
- PC 和手机端仅有纵向滚动，无横向滚动，所有表单控件完整可见。
